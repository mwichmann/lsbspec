Specification: LSB C++ generic
Specification: LSB C++ AMD64, IA64, PPC64, S390, S390X
Edition of Specification: 4.0
Type: Error
Status: Unapproved
Bug reference: http://bugs.linuxbase.org/show_bug.cgi?id=2950

Rationale: some constructors for std::ctype_byname<wchar_t>
and std::numpunct<char> were assigned to the incorrect reference
specification.  These are actually standard C++ classes and their
constructors are described in ISO C++.


In LSB-CXX-generic:

   In Table 9-151. libstdcxx - Class ctype_byname<wchar_t> Function Interfaces

Change:

   ctype_byname<wchar_t>::~ctype_byname()(GLIBCXX_3.4) [CXXABI]

To:

   ctype_byname<wchar_t>::~ctype_byname()(GLIBCXX_3.4) [ISOCXX]

===
